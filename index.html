📚 01 디지털 데이터 압축

1. 디지털 데이터 압축의 이해 (40쪽)

  • 압축의 개념: 한정된 저장 공간에 데이터를 효율적으로 기록하기 위해 저장할 데이터를 더 적은 수의 비트로 표현하는 기술입니다. 압축을 통해 저장 공간을 절약하고, 네트워크 전송 속도를 높여 디지털 탄소 중립을 실천할 수 있습니다.
  • 압축과 복원 과정: 데이터를 더 작은 크기의 특정 형식으로 변환하는 '인코딩(Encoding)' 과정과, 인코딩된 압축 데이터를 다시 원래 형태로 복원하는 '디코딩(Decoding)' 과정으로 이루어집니다.
  • 압축 방식의 분류:
    • 비손실 압축: 데이터 압축 후 복원했을 때 입력 데이터와 출력 데이터가 완벽히 동일한 방식입니다.
    • 손실 압축: 사람의 눈이나 귀로 잘 인식하지 못하는 정보를 제거하여 압축하므로, 복원했을 때 원래 데이터와 차이가 발생하지만 파일 크기를 획기적으로 줄일 수 있는 방식입니다.

2. 문자열 압축 (41쪽)

  • 문자열 압축은 데이터가 조금이라도 유실되면 의미가 달라지므로 반드시 비손실 압축 방식을 사용해야 합니다.
  • 런-렝스(Run-Length) 인코딩: 연속해서 반복되는 같은 문자를 '반복 문자 개수와 반복 문자의 쌍'으로 묶어서 표현하는 방식입니다. (예: AAAAA → 5A)
  • 렘펠-지브(Lempel-Ziv) 인코딩: 부분 문자열의 '상대적 위치와 패턴의 길이'로 반복 문자열을 치환하는 방식입니다. 떨어져 있는 같은 패턴도 압축할 수 있어 데이터가 많을수록 효율이 높습니다.

3. 이미지 압축 (42쪽)

  • BMP: 각 화소에 색 정보를 있는 그대로 기록하는 무압축 파일 형식으로, 고해상도 이미지를 표현하지만 용량이 매우 커서 인터넷 공유에 불리합니다.
  • 이미지 압축 파일 형식:
    • PNG: 비손실 압축 파일 형식으로, 품질 손실 없이 압축이 가능합니다.
    • JPEG / JPG: 사람 눈이 잘 인식하지 못하는 미세한 색상 차이를 제거하는 손실 압축 파일 형식입니다.
    • GIF: 256가지 이하의 색상만 지원하며 손실/비손실 압축이 혼용되는 파일 형식입니다.

4. 소리 압축 (44쪽)

  • WAV: 아날로그 신호를 그대로 디지털로 변환한 무압축 웨이브 파일 형식입니다. 샘플을 추출하는 간격이 좁고 비트 수가 많을수록 원음에 가까운 고음질 소리를 냅니다.
  • 소리 압축 파일 형식:
    • FLAC: 수학적 차이 등을 이용해 데이터 손실 없이 압축하는 비손실 압축 형식입니다.
    • MP3, AAC: 사람이 잘 듣지 못하는 주파수 대역의 신호를 과감히 잘라내어 용량을 줄이는 손실 압축 형식입니다.

🔑 02 디지털 데이터 암호화

1. 디지털 데이터 암호화의 이해 (49쪽)

  • 개념: 권한이 없는 사람이 데이터를 알아볼 수 없게 변환하는 기술로, 보안 위협을 줄이고 유출 시 복구에 소요되는 컴퓨팅 자원을 절약해 줍니다.
  • 암호화와 복호화: 누구나 읽을 수 있는 '평문'을 비밀스러운 '암호문'으로 바꾸는 과정을 암호화, 반대로 암호문을 원래의 평문으로 되돌리는 과정을 복호화라고 합니다.
  • 구성 요소: 평문, 암호문, 키(Key), 암호 알고리즘의 4가지 요소로 구성되며, 이 중 '키'는 평문을 암호문으로 바꿀 때 사용하는 핵심 문자열입니다.

2. 암호화 활용 사례 (50쪽)

  • 고대/근대 암호화 (주로 군사적 목적):
    • 전치 암호: 평문 문자의 '자리(위치)를 바꿔서' 암호문을 만드는 방식으로, 고대의 '스키테일 암호'가 대표적입니다.
    • 치환 암호: 평문을 '다른 문자로 대체시켜서' 암호문을 만드는 방식으로, 알파벳을 키값만큼 이동시키는 '카이사르 암호'나 근대 독일의 기계식 암호기인 '에니그마'가 있습니다.
  • 현대 암호화 (디지털 정보 보호 및 소유권 증명):
    • 대칭키 알고리즘: 송신자와 수신자가 암호화와 복호화에 '같은 키'를 사용하는 방식입니다.
    • 공개키 알고리즘: 모두에게 '공개된 키'로 암호화하고, 나만 아는 '개인키(비밀키)'로 복호화하는 방식입니다.
    • 해시 함수: 별도의 키 없이 일정한 규칙을 적용해 고정된 길이의 해시값을 출력하는 알고리즘으로, 패스워드 저장에 쓰입니다.
    • 블록체인과 NFT: 데이터를 분산된 블록에 담아 체인 형태로 연결하고 모든 사용자에게 공유하여 위조나 변조를 막는 기술이 블록체인입니다. 이를 기반으로 디지털 자산에 고윳값을 부여하여 소유권을 증명하는 기술이 NFT(대체 불가능 토큰)입니다.

📊 03 빅데이터 개념과 분석

1. 빅데이터 개념 이해 (55쪽)

  • 개념: 디지털 환경에서 생성되는 수십 테라바이트 이상의 대규모 데이터로, 데이터를 효과적으로 저장하고 데이터 과학 기술(통계, 기계학습 등)로 가치를 추출하여 활용하는 포괄적 개념입니다.
  • 빅데이터의 5가지 특징 (5V): 규모(Volume), 속도(Velocity), 다양성(Variety), 정확성(Veracity), 가치(Value)입니다.
  • 데이터의 유형:
    • 정형 데이터: 표, 스프레드시트, CSV 파일처럼 행과 열로 구분된 고정된 형식과 구조에 맞게 저장된 데이터입니다.
    • 비정형 데이터: 이미지, 소리, 사진, 동영상, SNS 텍스트 등 고정된 구조나 형태가 없는 데이터입니다.

2. 빅데이터 활용 분야 (56쪽)

  • 빅데이터는 기상 예측, 질병 및 유행병 예측, 맞춤형 헬스케어, 스마트 교통, 신용 카드 이상 거래 탐지, 맞춤형 상품 및 음악 추천 등 정치·사회·의료·보건·경제·산업 전반에 걸쳐 합리적 의사 결정을 돕는 데 쓰입니다.

3. 빅데이터 분석 (58쪽)

  • 빅데이터 분석은 데이터를 다양한 관점에서 관찰하고 이해하는 '탐색적 데이터 분석(EDA)' 방법을 주로 사용하며, 다음 4단계로 진행됩니다.
    • [1단계] 문제 정의하기: 분석의 목적, 필요성, 요구 사항을 파악합니다.
    • [2단계] 데이터 수집하기: 분석에 필요한 데이터를 자동화 수집기, 검색, 설문 등을 통해 수집합니다. 이때 저작권, 개인 정보 노출, 데이터 편향성, 데이터양의 균형을 반드시 고려해야 합니다.
    • [3단계] 데이터 분석하기 (전처리와 시각화):
      • 데이터 전처리: 평균, 최빈값 등 통계 지표를 확인하고 누락된 값인 결측치, 분포 범위를 크게 벗어난 이상치, 중복값 등을 찾아 분석에 적합하게 가공합니다.
      • 데이터 시각화: 데이터를 효과적으로 전달하기 위해 색채나 도형으로 표현하여 시간, 관계, 비교 분석을 진행합니다.
    • [4단계] 공유 및 평가하기: 분석 결과를 검토하고 평가합니다.

📚 Ⅲ 알고리즘과 프로그래밍

01 문제 분해와 모델링

  • 1. 문제 분석 (81쪽)
    • 문제 해결의 첫 단계로, 문제를 정확하게 이해하고 분석하는 과정입니다.
    • 파악해야 할 4가지 요소:
      • 현재 상태: 문제의 초기 상태.
      • 목표 상태: 문제 해결을 통해 얻고자 하는 결과나 상태.
      • 요구 사항: 문제가 궁극적으로 해결하고자 하는 것.
      • 제약 조건: 문제에 제시된 각종 데이터나 조건 등의 한계.
  • 2. 추상화 (82쪽)
    • 개념: 일상생활 속 크고 복잡한 문제를 단순화하는 과정입니다.
    • 추상화의 과정 (2단계):
      • ① 문제 분해: 복잡한 문제를 해결하기 쉽게 '작은 단위의 문제'로 나누는 과정입니다. 이때 문제 사이의 관계나 순서에 유의해야 하며, 분해된 작은 문제들의 결과를 종합해 전체 문제를 해결합니다.
      • ② 핵심 요소 추출: 문제 해결과 관련 없는 불필요한 요소를 과감히 제거하고, 꼭 필요한 핵심 요소만 추출합니다. 정보를 구분하는 기준은 '문제 해결의 목적'에 따라 달라집니다.
  • 3. 모델링 (85쪽)
    • 추출한 핵심 요소들을 이해하기 쉽게 구조화하거나 단순화하는 과정입니다.
    • 대표적인 모델링 구조 3가지:
      • 표(Table): 데이터를 가로축(행)과 세로축(열)으로 구조화하며, 조건 비교나 시간표, 성적 관리에 유리합니다.
      • 트리(Tree): 나무가 가지를 뻗는 모습처럼 데이터의 계층적 관계를 표현합니다. 임의의 노드에서 다른 노드로 가는 순환 구조(사이클)가 없습니다.
      • 그래프(Graph): 데이터 항목인 노드(Node)와 이들 사이의 흐름을 잇는 간선(Edge)으로 구성되며, 다양한 연결 관계(지하철 노선도 등)를 표현합니다.

02 알고리즘

  • 1. 알고리즘의 이해 (93쪽)
    • 알고리즘의 개념: 문제 해결을 위한 구체적인 방법이나 절차입니다.
    • 3대 제어 구조:
      • 순차 구조: 위에서 아래로 순서대로 명령 실행.
      • 선택 구조: 조건(참/거짓)에 따라 실행 경로가 달라짐.
      • 반복 구조: 조건에 따라 특정 명령을 반복적으로 실행.
    • 알고리즘 표현 방법: 특정한 형식 없는 자연어, 프로그래밍 언어를 흉내 낸 의사 코드, 정해진 도형 기호를 이용하는 순서도가 있습니다.
    • 효율적인 알고리즘의 조건: 수행 시간이 짧고(작업량 최소화), 기억 장소(메모리) 사용량이 적어야 합니다.
  • 2. 정렬 알고리즘 (97쪽)
    • 수많은 데이터를 오름차순 또는 내림차순 등 특정 기준에 따라 나열하는 과정입니다.
    • 버블 정렬: 인접한 두 개의 데이터를 차례대로 비교하여 크기가 맞지 않으면 자리를 교환합니다.
    • 선택 정렬: 정렬되지 않은 데이터 중에서 가장 작은 최솟값(또는 최댓값)을 찾아 기준 위치의 데이터와 교환합니다. 버블 정렬보다 교환 횟수가 적은 편입니다.
  • 3. 탐색 알고리즘 (102쪽)
    • 순차 탐색: 목록의 맨 앞부터 처음부터 끝까지 차례대로 비교하며 찾는 방법입니다. 데이터가 많을수록 탐색 시간이 선형적으로 늘어나 비효율적입니다.
    • 이진 탐색: 탐색 범위를 절반씩 줄여가며 찾는 방법으로, 사전에 데이터가 반드시 정렬되어 있어야만 사용할 수 있습니다.

03 프로그래밍의 이해

  • 1. 변수와 자료형 (110쪽)
    • 변수: 값을 저장하는 기억 공간입니다.
      • 변수명 규칙: 알파벳 대소문자 구분, 한글, 숫자, 밑줄(_) 사용 가능. 단, 숫자로 시작할 수 없고, 파이썬 예약어(if, for 등)는 사용할 수 없습니다.
    • 자료형: 저장될 데이터의 유형으로 수치형(정수 int, 실수 float), 불형(bool), 문자열(str)이 있습니다.
    • 연산자:
      • 산술 연산자: //(몫), %(나머지), **(거듭제곱) 등.
      • 비교 연산자: ==(같다), !=(다르지 않다) 등.
      • 논리 연산자: and(둘 다 참이어야 참), or(하나라도 참이면 참), not(부정).
  • 2. 표준입출력과 파일입출력 (114쪽)
    • 표준입출력: input() 함수로 키보드 입력을 받습니다. 입력값은 모두 문자열로 취급되므로 산술 연산을 하려면 int() 등으로 형 변환해야 합니다. 화면 출력은 print()를 사용합니다.
    • 파일입출력: 대용량 데이터를 영구적으로 처리하기 위해 사용합니다. open()으로 열고 반드시 close()로 닫아야 합니다.
      • 접근 방식: r(읽기), w(덮어쓰기), a(기존 내용 뒤에 추가하기).
  • 3. 다차원 데이터 구조 (119쪽)
    • 데이터를 행과 열, 여러 차원으로 저장하는 구조입니다.
    • 1차원 리스트: 데이터를 묶어 관리하며, 인덱스는 0부터 시작합니다. (주요 명령어: clear(), index(), count(), sum())
    • 2차원 리스트: 가로축(행)과 세로축(열)의 두 가지 인덱스로 값에 접근합니다 (예: 리스트명[행][열]). 영화관 좌석, 성적표 등에 활용됩니다.

04 제어 구조의 응용

  • 1. 조건문 (125쪽)
    • 주어진 조건에 따라 명령을 선택해 수행합니다. if (단순 조건문), if~else, if~elif~else (다중 조건문), 중첩 조건문이 있습니다.
    • 파이썬 문법 주의: 조건문 끝에는 반드시 콜론(:)을 써야 하며, 같은 블록 내의 명령문은 반드시 들여쓰기(Indentation)가 일치해야 합니다.
  • 2. 반복문 (129쪽)
    • for문: 특정 범위(range() 함수나 리스트 등) 안에서 원하는 횟수만큼 반복할 때 사용합니다.
    • while문: 주어진 조건식이 참(True)인 동안 계속 반복합니다. 조건에 True나 1을 쓰면 무한 루프가 됩니다.
    • 반복문 제어:
      • break: 반복문을 즉시 강제로 종료하고 빠져나옵니다.
      • continue: 현재 반복의 아래 코드를 건너뛰고 다음 반복의 처음으로 돌아가 계속 실행합니다.
  • 3. 제어 구조 프로그래밍 (135쪽)
    • 실생활 문제는 순차, 선택, 반복 구조를 복합적으로 중첩하여 사용해야 풀 수 있습니다.
    • random 모듈: 임의의 난수를 구하는 라이브러리로, random.randint(1, 5)는 1부터 5 사이의 임의의 정수를 반환합니다.

05 함수와 라이브러리

  • 1. 함수 (139쪽)
    • 함수: 특정 기능을 수행하는 코드의 묶음입니다.
    • 종류: 기본 제공되는 내장 함수(print, input 등)와, 사용자가 직접 def 키워드로 만드는 사용자 정의 함수가 있습니다.
    • 용어: 함수로 보내는 값은 인수, 그 값을 받는 변수를 매개 변수라 하며, 결과를 돌려줄 때는 반환값(return)을 사용합니다.
    • 지역 변수 vs 전역 변수: 특정 함수 안에서만 쓰이는 것은 지역 변수, 프로그램 전체에서 쓰이는 것은 전역 변수입니다. 함수 내부에서 전역 변수 값을 수정하려면 global 선언이 필요합니다.
  • 2. 모듈과 라이브러리 (144쪽)
    • 모듈: 함수, 변수, 클래스 등을 묶어 놓은 하나의 파이썬 파일(.py)입니다.
    • 라이브러리: 관련된 모듈들을 모아 놓은 패키지입니다.
    • 사용법: import 모듈명 [as 별명] 형태로 불러옵니다.
    • 외부 라이브러리: 별도 설치가 필요하며 데이터 분석에 핵심적인 역할을 합니다. (표 형태 데이터 조작: pandas, 수치 계산 배열: numpy, 그래프 시각화: matplotlib).

06 객체지향 프로그래밍

  • 1. 객체지향 프로그래밍의 이해 (153쪽)
    • 현실 세계의 사물을 객체로 추상화하고, 이들의 상호 작용으로 프로그램을 구현하는 방식입니다.
    • 객체: 고유한 속성(변수)과 기능(메소드)을 가진 주체입니다.
    • 클래스: 객체들을 생성하기 위한 공통된 설계도(틀)입니다.
    • 인스턴스: 클래스를 통해 실제로 생성된 각각의 개별 객체입니다.
  • 2. 객체지향 프로그래밍의 구현 (156쪽)
    • class 키워드로 클래스를 정의하며, 클래스 내부에 정의된 함수를 메소드(method)라고 부릅니다.
    • 인스턴스 생성: 인스턴스명 = 클래스명().
    • 클래스 내부의 메소드에는 매개 변수로 self를 반드시 포함해야 하며, 이는 인스턴스 자기 자신을 가리킵니다.

🤖 Ⅳ 인공지능

01 인공지능과 지능 에이전트

  • 1. 사람과 인공지능 (174쪽)
    • 인공지능(AI)의 개념: 인식, 학습, 추론, 문제 해결, 언어 구사 등 사람의 지능적인 판단과 결정을 컴퓨터 소프트웨어로 구현하는 것입니다.
      • 최근의 정의: 단순히 사람을 흉내 내는 것을 넘어, 주변 상황과 상호 작용하고 자율적이고 지능적인 판단으로 '주어진 목표를 이루는 지능 에이전트'의 관점에서 인공지능을 정의합니다.
    • 사람과 인공지능의 특징 비교:
      • 사람의 뇌: 새로운 것 만들기, 직관적인 판단, 번뜩이는 아이디어나 직감 등에 강합니다.
      • 인공지능: 빠르고 정확한 계산, 수많은 데이터의 기억, 탐색 및 논리적 추론 등에 강합니다.
  • 2. 에이전트와 지능 에이전트 (175쪽)
    • 에이전트(Agent) vs 지능 에이전트(Intelligent Agent)
      • 일반 에이전트: 누군가를 대신하여 환경과 상호 작용하며, '미리 설정된 규칙'에 따라서만 행동하는 시스템입니다. (예: 설정된 온도에 도달하면 꺼지는 단순 온도 조절기).
      • 지능 에이전트: 설정된 규칙에 얽매이지 않고, '학습과 추론'으로 스스로 행동을 개선하거나 새로운 규칙과 전략을 만들어 내어 상황에 효과적으로 대응하는 시스템입니다. (예: 주변 상황을 파악해 스스로 판단하는 로봇 청소기, 자율주행 자동차, 챗봇).
    • 지능 에이전트의 4가지 핵심 특성 (★시험 출제 1순위):
      • 자율성: 사람의 간섭 없이 사용자 대신 스스로 판단하고 행동합니다.
      • 목적성: 단순히 환경에 반응하는 것이 아니라, 주어진 목적을 달성하는 방향으로 행동합니다.
      • 적응성: 환경 변화나 사용자의 패턴에 따라 자신의 판단과 행동을 능동적으로 조정합니다.
      • 사회성: 지능 에이전트끼리 또는 사람과 데이터 및 정보를 주고받으며 상호 작용합니다.
    • 지능 에이전트의 동작 원리 (4단계):
      • ① 인식 (센서): 카메라, 마이크 등 센서로 주변 환경 데이터를 받아들입니다.
      • ② 상황 판단: 현재 상황(예: 바닥에 쓰레기가 있음)을 파악합니다.
      • ③ 행동 결정 (인공지능): 어떻게 할지(학습/추론) 결정합니다.
      • ④ 행동 (구동기/액추에이터): 모터, 스피커 등을 이용해 실제 물리적 행동을 취합니다.
    • 다중(Multi) 지능 에이전트: 개별 에이전트가 서로 경쟁하거나 협력하여 더 복잡한 목적을 달성하는 시스템입니다. (예: 로봇 축구, 여러 대가 협력하는 재난 구조 드론).

02 기계학습의 개념

  • 1. 기계학습의 이해 (180쪽)
    • 인공지능, 기계학습, 딥러닝의 포함 관계:
      • 인공지능(AI): 인간의 지적 능력을 컴퓨터로 구현한 가장 넓은 의미의 기술입니다.
      • 기계학습(Machine Learning): 인공지능의 한 분야로, 컴퓨터가 수많은 데이터로 학습하여 모델을 만들고, 이 모델을 바탕으로 스스로 규칙을 찾아 예측이나 분류를 하는 기술입니다.
      • 딥러닝(Deep Learning): 기계학습의 한 분야로, 인간의 뉴런과 비슷한 '인공 신경망' 구조를 사용하여 정보를 스스로 처리하고 학습하는 고도화된 기술입니다.
    • 기계학습의 필요성: 사람이 직접 명확한 규칙(알고리즘)을 설계하여 프로그래밍하기에는 너무 복잡하거나 뚜렷한 규칙을 찾기 어려운 문제를 해결할 때 반드시 필요합니다.
    • 기계학습의 실생활 활용 사례:
      • 북극 해빙 면적 예측: 인공위성 데이터로 학습하여 미래의 기후와 빙하 면적을 예측합니다 (회귀).
      • 추천 시스템: 사용자의 선호도와 구매 이력 데이터를 학습해 맞춤형 콘텐츠(도서, 음악, 영상 등)를 추천합니다.
      • 게임 AI: 이용자의 수준을 파악해 난이도를 지능적으로 조절하고 상호 작용합니다.
      • 이 외에도 스팸 메일 분류, 의료 영상 질병 진단, 신용 카드 부정 거래 탐지 등에 널리 쓰입니다.
